Case Study 03 — Network Services

DHCP Server Configuration

CategoryNetwork Services
DifficultyBeginner–Intermediate
OS UsedWindows Server 2008 R2

Scenario

The organization's network relied on manually assigned static IP addresses, which caused address conflicts and made onboarding new devices cumbersome. A DHCP server was deployed to automate IP address assignment across the 192.168.1.0/24 subnet.

Overview

This case study documents installing the DHCP Server role, configuring a scope for the internal network, setting scope options for gateway and DNS, authorizing the server in Active Directory, and verifying clients receive addresses. DHCP reservations were also configured for servers and printers that needed consistent IP addresses while still benefiting from centralized management.

Case Study Video

Implementation Steps

  1. Verified the server had a static IP address before installing the DHCP role
  2. Opened Server Manager > Add Roles > selected DHCP Server
  3. Completed the DHCP installation wizard, specifying DNS and domain name
  4. Opened DHCP Manager from Administrative Tools
  5. Right-clicked IPv4 > New Scope and configured the scope name and description
  6. Set the IP range: 192.168.1.100 – 192.168.1.200 with subnet mask 255.255.255.0
  7. Added exclusions for static devices: 192.168.1.1 – 192.168.1.99
  8. Set lease duration to 8 days for desktop workstations
  9. Configured Scope Options: 003 Router (192.168.1.1), 006 DNS (192.168.1.10), 015 Domain Name
  10. Activated the scope and authorized the DHCP server in Active Directory
  11. Created DHCP reservations for 3 printers and 2 servers using their MAC addresses
  12. Tested from a client: ran ipconfig /release and ipconfig /renew — client received IP from the scope

Key Findings

  • DHCP authorization in AD prevents rogue DHCP servers from distributing incorrect network settings
  • Exclusions should cover the full static IP range used by servers, routers, and infrastructure devices
  • DHCP Reservations provide static-like addressing with centralized management benefits
  • Scope Option 003 (Router) is critical — without it, clients cannot reach other subnets or the internet
  • Scope Option 006 (DNS Servers) should point to the domain controller's IP for AD environments
  • Short lease durations suit guest/wireless networks; longer leases reduce DHCP server load for desktops
  • The DHCP audit log at C:\Windows\System32\dhcp logs all lease activity for troubleshooting
  • In Windows Server 2012+, DHCP Failover allows two servers to share a scope — not available in 2008 R2

Outcome & Lessons Learned

After deploying DHCP, IP address conflicts dropped to zero and new device onboarding time was reduced significantly. Reservations ensured that critical infrastructure devices retained consistent addresses without manual static configuration on each device. The key lesson was that authorizing the DHCP server in Active Directory before activating the scope prevents a period where both old static assignments and new dynamic assignments could conflict.
