Case Study 05 — Storage

RAID Configuration (Striped, Mirrored, Spanned, RAID-5)

CategoryStorage Management
DifficultyIntermediate
OS UsedWindows Server 2008 R2

Scenario

The server hosting the organization's file shares used a single disk with no redundancy. A storage reconfiguration project was undertaken to implement software RAID using Windows Server's Disk Management, evaluating and demonstrating all four volume types: Spanned, Striped (RAID 0), Mirrored (RAID 1), and RAID-5.

Overview

This case study documents the creation and comparison of all four dynamic disk volume types available in Windows Server 2008 R2. Each volume type was created, tested for performance and fault tolerance characteristics, and evaluated for suitability in different use cases. The final production configuration used RAID-5 for the file share data volume and RAID-1 for the OS volume.

Case Study Video

Implementation Steps

  1. Added five additional virtual disks to the test server environment
  2. Opened Disk Management and converted all new disks to Dynamic Disks
  3. Created a Spanned Volume across 2 disks — data fills first disk before using the second
  4. Created a Striped Volume (RAID 0) across 2 disks — data striped for maximum read/write speed
  5. Created a Mirrored Volume (RAID 1) across 2 disks — identical copy maintained on both disks
  6. Created a RAID-5 Volume across 3 disks — striping with parity for balanced performance and fault tolerance
  7. Formatted all volumes as NTFS and wrote test data to each
  8. Simulated a disk failure on the RAID-1 volume by taking one disk offline
  9. Confirmed RAID-1 volume remained accessible in degraded mode during the failure
  10. Reintroduced the failed disk and confirmed RAID-1 resynchronized automatically
  11. Simulated a disk failure on the RAID-5 volume — confirmed continued operation with one disk missing
  12. Documented performance and capacity characteristics of each RAID type

Key Findings

  • Spanned Volume — combines multiple disks into one logical volume; no redundancy, no performance gain
  • Striped Volume (RAID 0) — highest performance; zero fault tolerance; losing one disk loses all data
  • Mirrored Volume (RAID 1) — full redundancy at 50% capacity efficiency; slight write performance overhead
  • RAID-5 — requires 3+ disks; tolerates one disk failure; usable capacity = (n-1) × disk size
  • Software RAID in Windows Server uses Dynamic Disks — incompatible with some third-party tools
  • Hardware RAID controllers offer better performance and have dedicated battery-backed cache
  • RAID protects against disk failure only — it does not protect against file corruption, deletion, or ransomware
  • Resynchronization of a degraded RAID array under load can temporarily impact server performance

Outcome & Lessons Learned

All four RAID types were successfully created and tested. RAID-1 was chosen for the OS volume and RAID-5 for the data volume in the final production configuration. The critical lesson reinforced was that RAID is not a backup strategy — despite RAID-5 surviving a disk failure, a separate backup solution was still required to protect against accidental deletion, ransomware, and multi-disk failure scenarios.
