Case Study 09 — OS Deployment

Windows Deployment Services (WDS)

CategorySystem Deployment
DifficultyAdvanced
OS UsedWindows Server 2008 R2

Scenario

The organization acquired 15 new workstations that needed Windows 7 installed and domain-joined before being distributed to staff. Manually installing from USB drives would take over a day. Windows Deployment Services was deployed to enable network-based OS installation, reducing the deployment time to under two hours for all 15 machines.

Overview

This case study documents the full WDS deployment pipeline: installing the role, importing boot and install images from a Windows 7 ISO, configuring PXE boot settings, and deploying the OS to bare-metal workstations over the network. An unattended answer file was created using Windows System Image Manager (WSIM) to automate the installation questions, enabling near-zero-touch deployment.

Case Study Video

Implementation Steps

  1. Verified prerequisites: DHCP, DNS, and AD DS were running and healthy
  2. Installed Windows Deployment Services role via Server Manager (both Deployment Server and Transport Server)
  3. Opened WDS console and configured the server: Integrated with Active Directory, image store at D:\RemoteInstall
  4. Set PXE response to: Respond to all known and unknown client computers
  5. Mounted the Windows 7 ISO and navigated to the sources folder
  6. Added boot.wim as a Boot Image in the WDS console
  7. Created an Image Group called 'Windows7' and added install.wim as an Install Image
  8. Selected 'Windows 7 Professional' edition from the WIM file
  9. Opened Windows System Image Manager (WSIM) and created an unattend.xml answer file
  10. Configured answer file to automate: disk partitioning, product key, regional settings, administrator password, domain join
  11. Associated the answer file with the Install Image in WDS console
  12. Configured first workstation to PXE boot: pressed F12 at startup, selected network boot
  13. Workstation received IP from DHCP, connected to WDS, and began downloading the boot image
  14. Windows 7 installed automatically using the answer file — no manual input required
  15. Confirmed workstation joined the domain and appeared in ADUC under the Computers OU

Key Findings

  • PXE Boot requires DHCP to provide the client with an IP address before WDS communication begins
  • If DHCP and WDS share a server, DHCP options 66 (Boot Server) and 67 (Bootfile Name) must be configured
  • The boot.wim boots a Windows PE environment over the network — it is separate from the OS being installed
  • Unattend.xml answer files created in WSIM eliminate all manual prompts during installation
  • WDS Multicast enables deploying one image to many clients simultaneously, reducing network load
  • Prestaged computers (registered by MAC address in AD) allow WDS to apply device-specific answer files
  • The WDS image store should be on a dedicated volume separate from the OS for performance and management
  • After deployment, MDT (Microsoft Deployment Toolkit) extends WDS with application installation and driver injection

Outcome & Lessons Learned

All 15 workstations were deployed with Windows 7 Professional and automatically domain-joined within two hours — a task that previously required a full day of manual effort. The unattended answer file was the critical enabler, eliminating the need for a technician to be present at each machine during installation. The key lesson was that the initial investment of time in creating and testing an accurate answer file pays dividends immediately at scale — a single well-tested answer file handles unlimited deployments without further intervention.
