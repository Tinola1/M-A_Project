Task 05 — Directory Services

Active Directory Domain Services

CategoryIdentity & Access Management
DifficultyIntermediate–Advanced
OS UsedWindows Server 2008 R2

Introduction

Active Directory Domain Services (AD DS) is Microsoft's directory service for managing users, computers, groups, and policies across an enterprise network. It is the cornerstone of Windows-based corporate IT infrastructure, enabling centralized authentication, authorization, and policy enforcement.

Overview

This task covers promoting a Windows Server to a Domain Controller by installing the AD DS role and running DCPromo, creating Organizational Units (OUs) for logical structure, creating and managing user accounts and security groups, and linking Group Policy Objects (GPOs) to enforce settings across the domain. Active Directory forms the foundation upon which DNS, DHCP, file services, and remote access all rely in a Windows domain environment.

Task Walkthrough Video

Step-by-Step Procedure

  1. Ensure DNS Server role is installed and a Forward Lookup Zone exists for your domain (e.g., lab.local)
  2. Open Server Manager > Add Roles > select 'Active Directory Domain Services'
  3. After installation, run DCPromo (Start > Run > dcpromo) to promote the server to a Domain Controller
  4. Choose 'Create a new domain in a new forest' for a fresh environment
  5. Enter the Fully Qualified Domain Name (FQDN) for the forest root domain (e.g., lab.local)
  6. Set the Forest Functional Level and Domain Functional Level to Windows Server 2008 R2
  7. Confirm DNS is installed and configured — DCPromo will install DNS if not already present
  8. Set the Directory Services Restore Mode (DSRM) password — store this securely
  9. Complete the wizard and reboot — the server is now a Domain Controller
  10. Open Active Directory Users and Computers (ADUC) from Administrative Tools
  11. Create Organizational Units (OUs): right-click the domain > New > Organizational Unit (e.g., IT, HR, Sales)
  12. Create User Accounts: right-click an OU > New > User — fill in name, logon name, and set a password
  13. Create Security Groups: right-click an OU > New > Group — assign users to groups for resource access
  14. Open Group Policy Management, right-click an OU, and link a new GPO to enforce desktop or security settings

Key Concepts

  • Domain Controller (DC) — a server running AD DS that authenticates users and enforces domain policies
  • Organizational Units (OUs) — containers for organizing users, computers, and groups logically
  • Global Catalog — a DC that holds a partial replica of all objects in the forest for fast queries
  • Kerberos — the default authentication protocol used by Active Directory for secure ticket-based auth
  • Group Policy Objects (GPOs) — collections of settings applied to OUs, sites, or the whole domain
  • SYSVOL — a shared folder on every DC that stores GPO files and logon scripts, replicated via DFS-R
  • FSMO Roles — five flexible single-master operations roles assigned to specific DCs for special functions
  • Trust Relationships — allow users in one domain to access resources in another domain or forest

Notes & Considerations

A domain requires at least two domain controllers for redundancy — a single DC represents a critical single point of failure. Always back up Active Directory using Windows Server Backup or a dedicated solution, paying particular attention to System State backups. Plan your OU structure carefully before deployment, as restructuring a live AD environment is disruptive. Group Policy processing follows the LSDOU order (Local, Site, Domain, OU) — understanding this hierarchy is essential for troubleshooting unexpected policy application.
