Task 06 — Network & Domain Management

DHCP Role & Joining Workstations in Domain

CategoryNetwork Services / Domain Management
DifficultyBeginner–Intermediate
OS UsedWindows Server 2008 R2

Introduction

DHCP (Dynamic Host Configuration Protocol) automates the assignment of IP addresses to network clients, eliminating the need to configure each device manually. Combined with joining workstations to an Active Directory domain, DHCP configuration is one of the first practical tasks performed when standing up a new Windows Server environment.

Overview

This task covers installing the DHCP Server role, creating an IP address scope for your network, configuring scope options (default gateway, DNS server, domain name), activating the scope, and authorizing the DHCP server in Active Directory. The second half of the task covers joining a Windows client workstation to the AD domain configured in the previous task, verifying the computer account appears in Active Directory, and logging in with a domain user account.

Step-by-Step Procedure

  1. Open Server Manager > Add Roles > select 'DHCP Server'
  2. In the DHCP wizard, specify the network connection bindings (the NIC that will serve DHCP)
  3. Enter the parent domain DNS name and the IP addresses of your DNS servers
  4. Create an IPv4 scope: provide a name, start IP, end IP, and subnet mask for the range
  5. Add exclusions for any static IPs in your range (servers, printers, network equipment)
  6. Set the lease duration — 8 days is typical for desktops; shorter for guest/wireless networks
  7. Configure Scope Options: 003 Router (default gateway), 006 DNS Servers, 015 DNS Domain Name
  8. Activate the scope — right-click the scope > Activate
  9. Authorize the DHCP server in Active Directory: right-click the server node > Authorize
  10. Verify clients receive IP addresses: on a client machine, run 'ipconfig /release' then 'ipconfig /renew'
  11. To join a workstation to the domain: right-click Computer > Properties > Change Settings > Change
  12. Select 'Domain', enter the domain name (e.g., lab.local), and provide Domain Admin credentials
  13. Reboot the workstation when prompted
  14. Log in with a domain account (DOMAIN\username) and verify the workstation appears in ADUC under Computers

Key Concepts

  • DHCP Scope defines the range of IP addresses available for dynamic assignment on a subnet
  • Scope Options configure additional network parameters sent to clients (gateway, DNS, domain)
  • DHCP Reservations bind a specific IP address to a client's MAC address for consistent addressing
  • DHCP Authorization in AD prevents rogue DHCP servers from distributing incorrect network settings
  • Lease Duration determines how long a client holds an assigned IP before needing to renew
  • DHCP Relay Agent forwards DHCP broadcasts across routers in multi-subnet environments
  • Domain Join creates a Computer Account object in Active Directory and establishes a secure channel
  • Group Policy is applied to the workstation upon domain join and at each subsequent logon/reboot

Notes & Considerations

For high availability, Windows Server 2012 and later support DHCP Failover — two DHCP servers can share a scope in either load-balancing or hot-standby mode. In Windows Server 2008 R2, the common approach is to split the scope between two servers (80/20 rule: primary server handles 80% of the range, secondary handles 20%). When joining workstations to a domain, ensure the client's preferred DNS server is pointing to the domain controller's IP address, not a public DNS server, otherwise the domain lookup will fail.
