Task 01 — Storage Configuration

RAID Setup

CategoryStorage Management
DifficultyIntermediate
OS UsedWindows Server 2008 R2

Introduction

RAID (Redundant Array of Independent Disks) is a method of combining multiple physical drives into a logical unit to improve performance, provide fault tolerance, or both. Configuring RAID is a fundamental skill in server administration, ensuring data availability even when individual disks fail.

Overview

This task covers the configuration of software RAID using Windows Server's built-in Disk Management and Storage tools. RAID levels such as RAID 0 (striping for speed), RAID 1 (mirroring for redundancy), and RAID 5 (striping with parity for balance) are explored. Understanding the differences between hardware and software RAID, and choosing the right level for a given workload, is essential for any systems administrator.

Task Walkthrough Video

Step-by-Step Procedure

  1. Open Server Manager and navigate to Storage > Disk Management
  2. Initialize all new disks by right-clicking each unallocated disk and selecting 'Initialize Disk'
  3. Choose GPT (GUID Partition Table) for disks larger than 2 TB, or MBR for legacy systems
  4. Right-click the first unallocated disk and select 'New Striped Volume' (RAID 0), 'New Mirrored Volume' (RAID 1), or 'New RAID-5 Volume'
  5. Add the required number of disks to the volume using the wizard (RAID 1 needs 2 disks; RAID 5 needs at least 3)
  6. Assign a drive letter and choose NTFS as the file system
  7. Format the volume and verify it appears as 'Healthy' in Disk Management
  8. Test fault tolerance by simulating a disk failure (for RAID 1/5) and confirming the volume remains accessible
  9. Reintroduce the failed disk and resynchronize the mirror or parity data

Key Concepts

  • RAID 0 — Striping: data split across drives for maximum speed, no redundancy
  • RAID 1 — Mirroring: identical data written to two drives, full redundancy at 50% capacity
  • RAID 5 — Striping with parity: requires 3+ drives, tolerates one drive failure with good read performance
  • RAID 10 — Mirrored stripes: combines RAID 1 and RAID 0 for speed and redundancy
  • Software RAID is managed by the OS, while hardware RAID uses a dedicated controller
  • Dynamic Disks are required for software RAID configurations in Windows Server
  • Resynchronization time depends on drive size and workload — plan maintenance windows accordingly
  • RAID is not a substitute for backups — it protects against hardware failure, not data corruption or deletion

Notes & Considerations

Always ensure your drives are identical in size when creating a RAID array, as the smallest disk in the set determines the usable capacity per drive slot. In production environments, hardware RAID controllers are preferred over software RAID for better performance and dedicated battery-backed cache. Document your RAID configuration thoroughly so that a replacement disk can be correctly integrated after a failure.
