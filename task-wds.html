Task 08 — OS Deployment

Windows Deployment Services

CategorySystem Deployment
DifficultyAdvanced
OS UsedWindows Server 2008 R2

Introduction

Windows Deployment Services (WDS) is a server role that enables network-based installation of Windows operating systems. Instead of manually installing Windows from a disc or USB on each machine, WDS allows technicians to boot a bare-metal computer over the network (PXE boot) and deploy a pre-configured Windows image to it automatically.

Overview

WDS works by hosting Windows images (WIM files) on a server and serving them to client machines that boot via PXE (Preboot Execution Environment) using network cards. The process requires an active DHCP server for IP assignment, a DNS server for name resolution, and the WDS role itself. WDS can serve both boot images (used to start the deployment environment) and install images (the actual Windows OS to be deployed). It integrates with Active Directory for authorization and can be configured with unattended answer files for zero-touch deployments.

Step-by-Step Procedure

  1. Ensure DHCP, DNS, and Active Directory Domain Services are already configured and running
  2. Open Server Manager > Add Roles > select 'Windows Deployment Services'
  3. Choose both role services: Deployment Server and Transport Server, then complete the installation
  4. Open the WDS console from Administrative Tools > Windows Deployment Services
  5. Right-click your server node and select 'Configure Server'
  6. Choose 'Integrated with Active Directory' and specify a path for the WDS image store (e.g., D:\RemoteInstall)
  7. Set the PXE server response: choose 'Respond to all client computers (known and unknown)'
  8. Right-click 'Boot Images' > Add Boot Image — browse to the Boot.wim file from a Windows installation ISO (sources\boot.wim)
  9. Right-click 'Install Images' > Add Install Image — create an image group, then add the Install.wim from the ISO (sources\install.wim)
  10. Select which Windows editions to add from the WIM file (e.g., Windows Server 2008 R2 Standard)
  11. On a client machine, configure the BIOS/UEFI to boot from Network (PXE) as the first boot device
  12. Power on the client — it will receive an IP from DHCP and connect to the WDS server
  13. Press F12 when prompted to initiate the network boot
  14. The Windows Deployment wizard will appear — select the install image and complete the installation

Key Concepts

  • PXE Boot — Preboot Execution Environment allows a computer to boot from a network server before loading a local OS
  • Boot Image (boot.wim) — a minimal Windows PE environment that loads the deployment wizard over the network
  • Install Image (install.wim) — the full Windows OS image deployed to the client's hard drive
  • WIM Format — Windows Imaging Format is a file-based disk image format that supports multiple editions in one file
  • Multicast Transmission — WDS can send one image to many clients simultaneously, reducing network bandwidth usage
  • Answer Files (unattend.xml) — XML files created with WSIM that automate the installation questions for zero-touch deployment
  • Image Groups — logical containers in WDS that organize install images and control access permissions
  • Prestaged Computers — machines pre-registered in AD by MAC address, allowing controlled PXE responses

Notes & Considerations

WDS requires that DHCP and WDS are either on separate servers, or if on the same server, specific DHCP options (60, 66, 67) must be configured to direct PXE clients to the WDS server. For large-scale deployments, consider pairing WDS with the Microsoft Deployment Toolkit (MDT) or System Center Configuration Manager (SCCM/MECM) for task sequencing, driver injection, and application deployment during OS installation. Always capture and test a reference image before deploying organization-wide to ensure all required software and settings are included.
